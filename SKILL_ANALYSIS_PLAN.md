# План разработки: система анализа знаний

Концепция и правила расчёта — в `SKILL_ANALYSIS.md`. Здесь — этапы кода и тесты.
Идём строго по порядку: этап не закрыт, пока не пройден его тест.

Принцип порядка: сначала чистая логика (её можно проверить автотестами без UI),
потом редактор (появляются данные), потом плеер (появляются события),
потом хранение и UI поверх готовых данных.

## Этап 0 — расчётное ядро + автотесты

Чистая функция без React и без сети — единственный этап с юнит-тестами
(логика каскада — самое ценное, что нельзя ломать молча).

**Код:**
- Подключить Vitest (`npm i -D vitest`, скрипт `"test": "vitest run"`).
- `src/shared/lib/skillScore.js`:
  - константы `THRESHOLD_SLOW = 15000`, `ATTEMPTS_OK = 1`;
  - `computePriority(events)` → `'high' | 'medium' | 'low' | null` — каскад из
    SKILL_ANALYSIS.md §5, события уже отфильтрованы по последней сессии;
  - `latestSessionEvents(allEvents)` — оставить события последней сессии
    каждого урока-источника;
  - `computeAllPriorities(allEvents)` → `Map<lessonId, priority>`.
- `src/shared/lib/skillScore.test.js` — кейсы:
  - пустая копилка → `null`;
  - `wrong` без `correct` → `high`;
  - `correct` поп.2 → `medium`; `correct` дольше порога → `medium`;
  - `dont_know` → `correct` → `medium`;
  - `know` → `correct` → `low`; `know` → `wrong` → `high`;
  - `correct` → `wrong` (свежее сильнее) → `high`;
  - только `know` → `low`;
  - две сессии: старая с `wrong`, новая чистая → берётся новая → `low`.

**Тест этапа:** `npm run test` — все кейсы зелёные. `npm run lint` чистый.

- [x] Этап 0 закрыт (16 тестов зелёные; общий `npm run lint` падает на старых
  файлах проекта, не связанных с этапом — новые файлы чистые)

## Этап 2 — сбор событий в плеере (пока только в память + дебаг-лог)

**Код:**
- `src/features/player/useAnswerStats.js` — хук:
  - `panelShown(nodeId)` — старт таймера (вызывается, когда панель стала
    кликабельной, после slide-up анимации);
  - `record(node, optionOrResult)` — собрать событие: `lessonId` (вариант →
    нода → ничего), `type` (из ✓ / из сигнала), `attempt` (счётчик по

- Вызовы из `LessonPlayer.jsx`: рядом с существующими `handleWordAnswer`,
  `handlePhraseAnswer`, `handlePhotoPick`; `panelShown` — из панелей по
  завершении анимации появления.
- События печатать в лог плеера (`pLog`) — попадут в скачиваемый дебаг-файл.

**Тест этапа (руками):**
1. Пройти стартовый урок по сценарию из SKILL_ANALYSIS.md §5 (с ошибками в to,
   «конечно знаю» про please).
2. Скачать дебаг-лог — сверить события: lessonId, type, attempt (складываются
   между нодами одного урока), timeMs (не включает аудио учителя — ответить

3. Нода без привязки не даёт события.

## Этап 4 — полоски приоритета на схеме уроков

**Код:**
