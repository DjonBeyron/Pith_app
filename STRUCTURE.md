# Карта файлов проекта

Что лежит в каждой папке и зачем нужен каждый файл. Простыми словами, без терминов.

**Правило:** если добавляешь новый файл — добавь о нём строку сюда. Если удаляешь файл —
удали его строку отсюда. Это правило обязательно, оно есть и в `CLAUDE.md`.

---

## Корень проекта

| Файл | Зачем нужен |
|------|-------------|
| `PROJECT.md` | Описание задачи: что строим, что уже решили, что ещё обсуждаем |
| `STRUCTURE.md` | Этот файл — карта всех файлов проекта |
| `CLAUDE.md` | Главные правила для Claude Code: как писать код в этом проекте |
| `README.md` | Создан автоматически при создании репозитория на GitHub, содержимого пока почти нет |
| `supabase_schema.sql` | Текст для настройки базы данных — таблицы и кто может их читать/писать |
| `.env.local` | Секретные ключи (пароли, токены) — здесь, чтобы их не было видно в коде |
| `.gitignore` | Список того, что не нужно сохранять в git (например, `.env.local`, `node_modules`) |
| `package.json` | Список программ-зависимостей и команды для запуска проекта (`npm run dev` и т.д.) |
| `package-lock.json` | Точные версии всех зависимостей — создаётся автоматически, не трогаем руками |
| `vite.config.js` | Настройки сборщика Vite (как собирать и запускать сайт) |
| `eslint.config.js` | Правила проверки кода — в т.ч. ограничение на длину файла |
| `index.html` | Самая первая HTML-страница, в которую подключается приложение |
| `start-dev.bat` | Двойной клик запускает сервер разработки (`npm run dev`) без открытия терминала вручную |

## `supabase/` — настройки серверной части

| Файл | Зачем нужен |
|------|-------------|
| `supabase/config.toml` | Настройки проекта Supabase (локальные порты и т.п.) |
| `supabase/functions/r2-upload-url/index.ts` | Серверная функция: выдаёт временную ссылку, по которой браузер может загрузить файл прямо в облачное хранилище (Cloudflare R2) |
| `supabase/functions/r2-upload-url/deno.json`, `.npmrc` | Технические настройки для этой функции |
| `supabase/functions/r2-delete/index.ts` | Серверная функция: удаляет файл из облачного хранилища по ссылке |
| `supabase/functions/r2-delete/deno.json` | Технические настройки для этой функции |
| `supabase/functions/transcribe-audio/index.ts` | Серверная функция: принимает аудиофайл (или R2 URL), транскрибирует через Groq Whisper, возвращает тайминги слов. Ключ GROQ_API_KEY хранится только здесь |
