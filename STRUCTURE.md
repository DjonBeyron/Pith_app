# Карта файлов проекта

Что лежит в каждой папке и зачем нужен каждый файл. Простыми словами, без терминов.

**Правило:** если добавляешь новый файл — добавь о нём строку сюда. Если удаляешь файл —
удали его строку отсюда. Это правило обязательно, оно есть и в `CLAUDE.md`.

---

## Корень проекта

| Файл | Зачем нужен |
|------|-------------|
| `PROJECT.md` | Описание задачи: что строим, что уже решили, что ещё обсуждаем |
| `STRUCTURE.md` | Этот файл — карта всех файлов проекта |
| `CLAUDE.md` | Главные правила для Claude Code: как писать код в этом проекте |
| `README.md` | Создан автоматически при создании репозитория на GitHub, содержимого пока почти нет |
| `supabase_schema.sql` | Текст для настройки базы данных — таблицы и кто может их читать/писать |
| `.env.local` | Секретные ключи (пароли, токены) — здесь, чтобы их не было видно в коде |
| `.gitignore` | Список того, что не нужно сохранять в git (например, `.env.local`, `node_modules`) |
| `package.json` | Список программ-зависимостей и команды для запуска проекта (`npm run dev` и т.д.) |
| `package-lock.json` | Точные версии всех зависимостей — создаётся автоматически, не трогаем руками |
| `vite.config.js` | Настройки сборщика Vite (как собирать и запускать сайт) |
| `eslint.config.js` | Правила проверки кода — в т.ч. ограничение на длину файла |
| `index.html` | Самая первая HTML-страница, в которую подключается приложение |
| `start-dev.bat` | Двойной клик запускает сервер разработки (`npm run dev`) без открытия терминала вручную |

## `supabase/` — настройки серверной части

| Файл | Зачем нужен |
|------|-------------|
| `supabase/config.toml` | Настройки проекта Supabase (локальные порты и т.п.) |
| `supabase/functions/r2-upload-url/index.ts` | Серверная функция: выдаёт временную ссылку, по которой браузер может загрузить файл прямо в облачное хранилище (Cloudflare R2) |
| `supabase/functions/r2-upload-url/deno.json`, `.npmrc` | Технические настройки для этой функции |
| `supabase/functions/r2-delete/index.ts` | Серверная функция: удаляет файл из облачного хранилища по ссылке |
| `supabase/functions/r2-delete/deno.json` | Технические настройки для этой функции |

## `src/` — код приложения

### `src/main.jsx`
Точка входа — самый первый файл, который запускается в браузере. Просто подключает `App`.

### `src/index.css`
Точка входа стилей — только список `@import` для всех css-файлов.

### `src/features/user/` — всё для вкладки «Пользователь»
| Файл | Зачем нужен |
|------|-------------|
| `UserTab.jsx` | Показывает файлы по одному каждые 3 сек (имитация чата), номер по порядку загрузки, проигрывает фото/видео/аудио; выгруженное видео показывает как стоп-кадр с кнопкой «Загрузить», клик — подгружает заново |
| `useSequentialPreload.js` | Качает файлы по одному строго по очереди (не параллельно, не дальше чем на несколько сообщений вперёд от показанного — см. `allowUpTo`), с авто-повтором при обрыве; держит в памяти буфер из 5 видео/аудио, при переполнении выгружает того, кто **дальше всех** от текущей позиции в чате (не того, кто дольше всех висит), фото никогда не выгружает, видео при выгрузке оставляет один кадр-превью; умеет подгрузить файл повторно по требованию |

### `src/shared/lib/` — вспомогательные функции без интерфейса
| Файл | Зачем нужен |
|------|-------------|
| `r2.js` | Загрузить файл в облако (`uploadToR2`) и удалить файл из облака (`deleteFromR2`) |
| `filesApi.js` | Получить список файлов из базы (`listFiles`), добавить файл в базу (`insertFile`), удалить запись из базы (`deleteFileRow`), посчитать размер файла в КБ/МБ (`formatBytes`), определить тип файла — фото/видео/аудио (`getMediaKind`) |
| `localFileStore.js` | Запасной локальный склад для файлов в браузере (IndexedDB) — пока не используется в интерфейсе, оставлен на будущее |
| `lessonsApi.js` | CRUD уроков в Supabase: получить список, создать, удалить, сохранить/загрузить граф (`script`) |
| `debug.js` | Включает/выключает подробные логи в консоли браузера (кнопка «Активировать дебаг») |
| `videoFrame.js` | Достаёт один кадр из видео как картинку — используется для превью выгруженного из памяти видео |
| `version.js` | Номер версии приложения (`APP_VERSION`) — показывается мелко наверху экрана |
| `textHighlight.js` | Утилиты для работы с выделениями: `buildCharStyles` (карта символ→стиль) и `hexToRgba` |
