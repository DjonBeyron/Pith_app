# Pithy App — Roadmap: система прогресса и XP

## Концепция платформы

Один учитель-автор создаёт контент. Студенты приходят сами или по ссылке.
Вся платформа выдержана в метафоре чата — ни один экран не выходит за её рамки.

### Два пути студента

```
НЕЗАРЕГИСТРИРОВАННЫЙ
  Ссылка / каталог → [Вводный урок] → диагностика слабых мест
                                            ↓
                          [Граф модуля] → [Тренажёры] → [Босс]
                                                             ↓
                                            Gate: зарегистрируйся
                                            чтобы сохранить прогресс
                                                             ↓
                                            (прогресс из localStorage
                                             переносится в Supabase)

ЗАРЕГИСТРИРОВАННЫЙ
  Вкладка [Все модули] → фильтр → выбирает любой → [Вводный] → ...
```

### Глобальный список (замена «карты»)

Выглядит как список чатов (Telegram-стиль). Доступен всем — без регистрации.
Незарегистрированный видит свой прогресс из localStorage прямо в списке.

```
┌─────────────────────────────────────┐
│ 🟠 I'm trying to please both        │
│    ████████░░  3/5 уроков · Продолжить│
├─────────────────────────────────────┤
│ 🔵 Can you help me out?             │
│    ██░░░░░░░░  1/4 · В процессе     │
├─────────────────────────────────────┤
│ ⬜ I'll get back to you             │
│    A2 · present continuous · 5 уроков│
├─────────────────────────────────────┤
│ ✅ Nice to meet you                 │
│    Пройдено · +120 XP заработано    │
└─────────────────────────────────────┘
```

Статусы строки: в процессе / новый / завершён (затемнён, можно повторить).

### Поиск и фильтр

Студент может найти модуль по:
- **Слову** — `trying`, `please`, `get` (парсится автоматически из текста фразы)
- **Конструкции** — `be + V-ing`, `to + infinitive` (добавляет учитель)
- **Правилу** — `present continuous`, `phrasal verb` (добавляет учитель)
- **Уровню** — A1 / A2 / B1 / B2 / C1 (выбирает учитель)

Теги хранятся в `phrase_modules` и добавляются вручную через админ-интерфейс.

## Концепция модуля

Приложение построено вокруг фраз (например «I'm trying to please both»).
Каждая фраза — это **модуль** из нескольких уроков:

```
Глобальная карта (все фразы-модули)
  └── Модуль «I'm trying to please both»
        ├── Вводный урок       → +1 уровень, +1 unlock-очко
        ├── Тренажёр "trying"  → +XP, +1 уровень, +1 очко
        ├── Тренажёр "please"  → +XP, +1 уровень, +1 очко
        ├── Тренажёр "both"    → +XP, +1 уровень, +1 очко
        └── Босс-урок          → большой XP, модуль завершён → глобальная карта
```

**XP** накапливается внутри модуля (word_choice + phrase_assembly дают по 10 XP за верный ответ).
Когда XP достигает порога — прогресс-бар босса заполняется и замок снимается.
Все XP вкладываются в замок босса и обнуляются — остаются только **уровни** (глобальные).

## 🔄 Фаза 1 — Регистрация (В РАЗРАБОТКЕ)

Прогресс переносится в Supabase без повторного прохождения.

## Ключевые правила XP

| Правило | Детали |
|---|---|
| Источники XP | `word_choice` + `phrase_assembly` (верный ответ) |
| Значение по умолчанию | 10 XP за ответ (настраивается в модуле) |
| Защита от фарма | UNIQUE(user_id, lesson_id, node_id) — только первое прохождение |
| 3-я попытка в сборке фразы | XP начисляется (ответ подсказан, но засчитывается) |
| Повтор урока | Разрешён (для практики), нода показывается как «пройдено», XP не начисляется повторно |
| XP при удалении ноды учителем | Сохраняется у пользователя |
| Уровни | Глобальные, +1 за каждый завершённый урок |
| XP vs уровни | XP тратится на разблокировку босса и уходит; уровни остаются навсегда |
| Офлайн | Уроки только онлайн |

## Диагностика вводного урока → цветные ветки

Вводный урок собирает данные об ошибках и скипах пользователя.
На основе этого карта модуля окрашивает ветки — фраза буквально **становится графом**.

### Что собираем

| Событие | Поле |
|---|---|
| Пролистал аудио/текст быстро (< 3 сек) | `skipped: true` |
| Ошибся в word_choice | `error: true, node_id` |
| Ошибся 2+ раз в phrase_assembly | `error_count: 2` |
| Верно с первой, не пропускал | `strong: true` |

Хранится в таблице `lesson_events` рядом с `xp_events`.

### Три ветки — три цвета

```
[ВВОДНЫЙ]
    │
    ├──🟠── I'm · trying   ← пропустил объяснение + ошибка
    │                         авто-открывается первым (первое очко тратится автоматически)
    │
    ├──🔵── to · please    ← были ошибки, но объяснение смотрел
    │                         рекомендуется вторым (значок «стоит пройти»)
    │
    ├──🟢── both           ← верно с первой, не пропускал
    │                         можно пропустить или взять бонусный XP
    │
    └──🔒── [БОСС]         ← заперт, прогресс-бар XP снизу
```

### Правила цветов

| Цвет | Условие | Поведение |
|---|---|---|
| 🟠 Оранжевый | ошибка + скип объяснения | авто-разблокируется первым unlock-очком |
| 🔵 Синий | ошибка ИЛИ скип (не оба) | значок «рекомендуем» на ноде |
| 🟢 Зелёный | верно с первой, не скипал | бонус +5 XP если пройдёт |
| ⚪ Серый | нет диагностических данных | нейтрально |

### Группировка слов в ветки

Несколько слов могут составлять одну ноду-ветку (например «I'm · trying» — оба оранжевые → одна нода).
Учитель задаёт группировку в редакторе модуля при расстановке тренажёров на canvas.
