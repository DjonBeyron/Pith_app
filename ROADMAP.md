# Pithy App — Roadmap: система прогресса и XP

## Концепция платформы

Один учитель-автор создаёт контент. Студенты приходят сами или по ссылке.
Вся платформа выдержана в метафоре чата — ни один экран не выходит за её рамки.

### Два пути студента

```
НЕЗАРЕГИСТРИРОВАННЫЙ
  Ссылка / каталог → [Вводный урок] → диагностика слабых мест
                                            ↓
                                            Gate: зарегистрируйся
                                            чтобы сохранить прогресс
                                             (прогресс из localStorage
                                             переносится в Supabase)
                                            ↓
                          [Граф модуля] → [Тренажёры] → [Босс]
                                                             
                                            
                                                             
                                           

ЗАРЕГИСТРИРОВАННЫЙ видит
  Вкладка [Все модули] → фильтр → выбирает любой → [Вводный] → ...
```

### Глобальный список (замена «карты»)

Выглядит как список чатов (Telegram-стиль). Доступен всем — без регистрации.
Незарегистрированный видит свой прогресс из localStorage прямо в списке.

```
┌─────────────────────────────────────┐
│ 🟠 I'm trying to please both        │
│    ████████░░  3/5 уроков · Продолжить│
├─────────────────────────────────────┤
│ 🔵 Can you help me out?             │
│    ██░░░░░░░░  1/4 · В процессе     │
├─────────────────────────────────────┤
│ ⬜ I'll get back to you             │
│    A2 · present continuous · 5 уроков│
├─────────────────────────────────────┤
│ ✅ Nice to meet you                 │
│    Пройдено · +120 XP заработано    │
└─────────────────────────────────────┘
```

Статусы строки: в процессе / новый / завершён (затемнён, можно повторить).

### Поиск и фильтр

Студент может найти модуль по:
- **Слову** — `trying`, `please`, `get` (парсится автоматически из текста фразы)
- **Конструкции** — `be + V-ing`, `to + infinitive` (добавляет учитель)
- **Правилу** — `present continuous`, `phrasal verb` (добавляет учитель)
- **Уровню** — A1 / A2 / B1 / B2 / C1 (выбирает учитель)

Теги хранятся в `phrase_modules` и добавляются вручную через админ-интерфейс.

## Концепция модуля

Приложение построено вокруг фраз (например «I'm trying to please both»).
Каждая фраза — это **модуль** с картой слов.

### Глобальная схема (путь пользователя)

```
[Список фраз — Telegram-стиль]
        │
        ▼
[СТАРТОВЫЙ УРОК фразы]  ← единственная точка входа в модуль
        │
        │  в конце: если не зарегистрирован → gate «зарегистрируйся»
        │
        ▼
[КАРТА МОДУЛЯ — слова фразы]
  ┌──────────────────────────────┐
  │  🟠 i'm      [████░░] 40 XP │  ← оранжевые открыты сразу
  │  🟠 trying   [██░░░░] 80 XP │
  │  🔵 to       [░░░░░░] 60 XP │  ← синие: кнопка «Открыть»
  │  🔵 please   [░░░░░░] 80 XP │
  │  🟢 bouh     [░░░░░░] 40 XP │  ← зелёные: кнопка «Открыть» (бонус)
  │                              │
  │  🔒 ФИНАЛ   [████████] XP   │  ← замок снимается после оранжевых+синих
  └──────────────────────────────┘
        │
        ▼ после финала
[Список фраз]  ← другие фразы, разбитые по тому же принципу
```

Из карты в любой момент можно вернуться в список (кнопка сверху ≡).
Попасть в карту можно только через стартовый урок — или повторно нажав на фразу в списке.

### XP — распределение по словам

Общий пул модуля = **1000 XP** (фиксировано).
Распределяется **автоматически** по сложности диагностики:

| Цвет | Вес | Пример (5 слов, веса 3/3/2/2/1 = 11) |
|------|-----|---------------------------------------|
| 🟠 Оранжевый | 3 | 3/11 × 1000 = **273 XP** |
| 🔵 Синий | 2 | 2/11 × 1000 = **182 XP** |
| 🟢 Зелёный | 1 | 1/11 × 1000 = **91 XP** |

Финал получает фиксированные **200 XP** (вычитаются из пула до распределения):
реальный пул слов = 800 XP, финал = 200 XP.

Формула расчёта — на клиенте при загрузке карты модуля, сохранять в БД не нужно.

### Как открывается слово-модуль

- 🟠 **Оранжевые** — открыты автоматически после стартового урока
- 🔵 **Синие / 🟢 Зелёные** — заблокированы, есть кнопка **«Открыть»**
  - При нажатии: накопленный XP пользователя «переливается» в прогресс-бар этого слова
  - Визуально: анимация перетекания XP → прогресс-бар заполняется → замок снимается
  - XP списывается с общего баланса пользователя

### Финал (ситуативный тест)

Финал = урок-ситуация (например «заказ пиццы на английском»).
Для учителя всегда разблокирован (`isTeacher` → замок снят).

**Замок снимается когда:**
- Все 🟠 оранжевые слова пройдены ✅
- Все 🔵 синие слова пройдены ✅
- Зелёные — по желанию, бонусный XP

**Прогресс-бар финала** = сумма XP оранжевых + синих / их суммарный XP-порог.

### Диагностика → цвета (живой статус)

Цвет слова НЕ статичный — обновляется после прохождения тренажёра:
- Прошёл оранжевый/синий без ошибок → становится 🟢
- Цвет = текущий уровень владения словом, не только первичная диагностика

**Холодный старт:** карты модуля не существует до прохождения стартового урока.
Стартовый урок — единственная точка входа, он и строит карту.

## ✅ Фаза 1 — Роутинг + Регистрация (ЧАСТИЧНО ГОТОВО)

Любой урок проходится без аккаунта. Прогресс живёт в localStorage (`pithy_guest`).
После вводного урока — gate «зарегистрируйся чтобы сохранить прогресс».
Прогресс переносится в Supabase без повторного прохождения.
Глобальный список доступен всем без регистрации.

### Роль учителя

Учитель — один hardcoded пользователь. Определяется через env:
```
VITE_TEACHER_EMAIL=teacher@pithy.app
```
```js
export const isTeacher = (user) => user?.email === import.meta.env.VITE_TEACHER_EMAIL
```
Учитель видит Canvas / Admin, студент — нет. Никаких ролей в базе не нужно.

### URL-роутинг

Сейчас в `App.jsx` — только `useState('lessons')`, URL всегда `/`.
Для работы ссылок нужен React Router v6:

```bash
npm install react-router-dom
```

Структура роутов:
```
/                    → ModuleList (глобальный список) — публичный
/module/:moduleId    → ModuleMap (граф фразы) — публичный
/lesson/:lessonId    → LessonPlayer — публичный
/canvas/:lessonId    → CanvasPage — только isTeacher
/admin               → AdminTab — только isTeacher
/login               → LoginScreen — публичный
```

`vercel.json` — обязателен для SPA (иначе прямые ссылки дают 404):
```json
{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
```

`App.jsx` переписывается с tab-state на `<BrowserRouter><Routes>`.
Текущие вкладки Уроки / Админ / Настройки становятся роутами учителя.

### localStorage — схема прогресса студента

Существующие ключи (не трогаем): `lesson_teacher_{id}`, `lesson_files_{id}`,
`hl_opacity`, `hl_recent`, `pithy_debug`.

Новый ключ `pithy_guest`:
```js
{
  completedNodes: {
    // [lessonId]: { [nodeId]: { xp: 10, completedAt: timestamp } }
    "lesson-abc": {
      "node-1": { xp: 10, completedAt: 1719400000 }
    }
  },
  diagnostics: {
    // [lessonId]: { [nodeId]: { skipped, errorCount, timeSpent (ms) } }
    "lesson-abc": {
      "node-2": { skipped: true, errorCount: 0, timeSpent: 1200 },
      "node-4": { skipped: false, errorCount: 2, timeSpent: 8400 }
    }
  },
  moduleProgress: {
    // [moduleId]: агрегированное состояние
    "module-xyz": {
      introCompleted: true,
      xpEarned: 30,
      unlockPoints: 1,
      unlockedLessonIds: ["lesson-def"],
      completedLessonIds: ["lesson-abc"]
    }
  }
}
```

После регистрации: батч-INSERT `completedNodes` → `xp_events`,
`moduleProgress` → `user_module_progress`, `diagnostics` → `lesson_events`.
Ключ `pithy_guest` очищается.

## Фаза 3 — XP в Supabase

**Supabase таблицы:**
```sql
xp_events (
  id, user_id, lesson_id, node_id, xp_earned, created_at,
  UNIQUE(user_id, lesson_id, node_id)   -- защита от фарма
)
lesson_events (                          -- диагностика вводного урока
  id, user_id, lesson_id, node_id,
  skipped BOOLEAN, error_count INT, time_spent_ms INT,
  created_at,
  UNIQUE(user_id, lesson_id, node_id)
)
user_module_progress (
  user_id, module_id,                   -- PK
  xp_earned INT DEFAULT 0,              -- пересчитывается триггером
  intro_done BOOLEAN DEFAULT FALSE,
  boss_done BOOLEAN DEFAULT FALSE,
  unlock_points INT DEFAULT 0,
  unlocked_lesson_ids UUID[] DEFAULT '{}',
  completed_lesson_ids UUID[] DEFAULT '{}'
)
user_global (
  user_id PK,
  level INT DEFAULT 0,                  -- пересчитывается триггером
  completed_module_ids UUID[] DEFAULT '{}'
)
```

**RLS-политики (Row Level Security):**
```sql
-- xp_events: юзер читает/пишет только свои
-- lesson_events: то же
-- user_module_progress: только свои строки
-- phrase_modules: публичное чтение (SELECT), запись — только TEACHER_EMAIL через service role
-- user_global: только своя строка
```

**Пересчёт XP — Supabase Trigger (не клиент):**
```sql
-- При INSERT в xp_events → обновить user_module_progress.xp_earned
-- При обновлении user_module_progress.intro_done → +1 к user_global.level
-- Клиент не считает XP — только пишет события
```

**Задачи:**
- [ ] Создать все 4 таблицы в Supabase
- [ ] Настроить RLS для каждой таблицы
- [ ] Написать триггер пересчёта `xp_earned` и `level`
- [ ] Батч-запись `xp_events` после завершения урока (`ON CONFLICT DO NOTHING`)
- [ ] Батч-запись `lesson_events` с диагностикой (из `pithy_guest.diagnostics`)
- [ ] Чтение прогресса при логине → синхронизация с `pithy_guest`
- [ ] XP за удалённые ноды сохраняется (триггер не пересчитывает ретроактивно)

## Ключевые правила XP

| Правило | Детали |
|---|---|
| Источники XP | `word_choice` + `phrase_assembly` (верный ответ) |
| Пул модуля | 1000 XP фиксировано. 200 XP — финал, 800 XP — слова |
| Распределение по словам | Автоматически по весу: 🟠 вес 3 / 🔵 вес 2 / 🟢 вес 1 |
| Защита от фарма | UNIQUE(user_id, lesson_id, node_id) — только первое прохождение |
| 3-я попытка в сборке фразы | XP начисляется (ответ подсказан, но засчитывается) |
| Повтор урока | Разрешён (для практики), XP не начисляется повторно |
| XP при удалении ноды учителем | Сохраняется у пользователя |
| Офлайн | Уроки только онлайн |

## Уровни и рулетка

### Пороги уровней

Уровень = глобальный, привязан к профилю пользователя (`user_profiles.level`).
Накапливается из **суммарного XP всех модулей** (XP не обнуляется после трат на разблокировку).

```
Уровень 1  →  100 XP
Уровень 2  →  200 XP  (накоплено 300 всего)
Уровень 3  →  300 XP  (накоплено 600 всего)
Уровень N  →  N × 100 XP сверх предыдущего
```

При достижении нового уровня → запускается **рулетка**.

### Рулетка

Всегда выигрыш — вопрос только в редкости приза.
Первая рулетка (уровень 1) — гарантированно «редкая» (wow-эффект с порога).

**Редкость:**
```
⬜ Обычное    70%
🔵 Редкое     20%
🟠 Эпическое   8%
👑 Легенда     2%
```

**Призы (предварительно — детали прорабатываются позже):**

| Редкость | Варианты |
|----------|----------|
| ⬜ Обычное | +50 XP бонус, бейдж-эмодзи (🔥 ⚡ 🎯), цвет ника |
| 🔵 Редкое | x2 XP на следующий урок, рамка аватара, пропуск замка слова |
| 🟠 Эпическое | анимированная рамка, подсказка в финале, «второй шанс» |
| 👑 Легенда | уникальный бейдж навсегда, ранний доступ к новой фразе |

**Механика дофамина:**
- Анимация замедления барабана — мозг «болеет» за результат
- «Почти легенда» — стрелка иногда проходит мимо 👑 на 1 деление
- Streak-бонус: 3 уровня подряд без пропуска уроков → шанс легенды ×2
- Публичность: системное сообщение в чате «🎉 [Имя] получил Золотую рамку»
- Дубль приза → конвертируется в +25 XP автоматически

**Детали рулетки прорабатываются в отдельной сессии.**

## Диагностика вводного урока → цветные ветки

Вводный урок собирает данные об ошибках и скипах пользователя.
На основе этого карта модуля окрашивает ветки — фраза буквально **становится графом**.

### Что собираем

| Событие | Поле |
|---|---|
| Пролистал аудио/текст быстро (< 3 сек) | `skipped: true` |
| Ошибся в word_choice | `error: true, node_id` |
| Ошибся 2+ раз в phrase_assembly | `error_count: 2` |
| Верно с первой, не пропускал | `strong: true` |

Хранится в таблице `lesson_events` рядом с `xp_events`.

### Три ветки — три цвета

```
[ВВОДНЫЙ]
    │
    ├──🟠── I'm · trying   ← пропустил объяснение + ошибка
    │                         авто-открывается первым (первое очко тратится автоматически)
    │
    ├──🔵── to · please    ← были ошибки, но объяснение смотрел
    │                         рекомендуется вторым (значок «стоит пройти»)
    │
    ├──🟢── both           ← верно с первой, не пропускал
    │                         можно пропустить или взять бонусный XP
    │
    └──🔒── [БОСС]         ← заперт, прогресс-бар XP снизу
```

### Правила цветов

| Цвет | Условие | Поведение |
|---|---|---|
| 🟠 Оранжевый | ошибка + скип объяснения | авто-разблокируется первым unlock-очком |
| 🔵 Синий | ошибка ИЛИ скип (не оба) | значок «рекомендуем» на ноде |
| 🟢 Зелёный | верно с первой, не скипал | бонус +5 XP если пройдёт |
| ⚪ Серый | нет диагностических данных | нейтрально |

### Группировка слов в ветки

Несколько слов могут составлять одну ноду-ветку (например «I'm · trying» — оба оранжевые → одна нода).
Учитель задаёт группировку в редакторе модуля при расстановке тренажёров на canvas.
